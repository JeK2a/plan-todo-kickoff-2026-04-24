# Acceptance Criteria для v1.0.0

## Цель

Зафиксировать измеримые критерии приемки первого релиза TODO MVP.

## Критерии по функциональности

### AC-01: Создание задачи

- Пользователь вводит непустой текст задачи и нажимает "Добавить".
- Новая задача появляется первой в списке.
- Пустая строка или строка из пробелов не создает новую задачу.

### AC-02: Изменение статуса

- Для каждой задачи доступно действие "Готово/Вернуть".
- После переключения задача меняет статус `done`.
- Визуально выполненная задача отображается зачеркнутой.

### AC-03: Удаление задачи

- Для каждой задачи доступно действие "Удалить".
- После удаления задача исчезает из списка без перезагрузки страницы.

### AC-04: Фильтрация

- Фильтр "Все" показывает все задачи.
- Фильтр "Активные" показывает только задачи с `done = false`.
- Фильтр "Выполненные" показывает только задачи с `done = true`.

### AC-05: Очистка выполненных

- Кнопка "Очистить выполненные" удаляет только задачи с `done = true`.
- Активные задачи остаются в списке.

### AC-06: Сохранение состояния

- После перезагрузки страницы список задач и их статусы восстанавливаются из `localStorage`.
- При отсутствии данных в `localStorage` приложение стартует с пустым списком.

## Критерии по качеству

### AC-07: Unit-тесты

- Команда `npm test` завершается с кодом `0`.
- Покрыты тестами операции:
  - фильтрация,
  - добавление,
  - переключение статуса,
  - удаление,
  - очистка выполненных,
  - подсчет выполненных.

### AC-08: CI-пайплайн

- Workflow CI запускается на `push` и `pull_request` в `master`.
- CI завершается успешно на актуальной версии кода.

## Definition of Done для релиза v1.0.0

Релиз считается принятым, если выполнены AC-01...AC-08 и соответствующие задачи
в GitHub Issues и GitHub Project переведены в статус `Done`.
