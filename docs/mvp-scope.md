# MVP Scope и не-цели

## Контекст

Этот документ фиксирует границы первого релиза `v1.0.0` для TODO MVP.
Цель релиза: дать пользователю рабочий минимальный сценарий управления задачами в браузере.

## Входит в MVP (Scope)

- Одностраничный интерфейс TODO.
- Добавление новой задачи через форму.
- Отображение списка задач.
- Переключение статуса задачи (активная/выполненная).
- Удаление отдельной задачи.
- Фильтрация задач: все, активные, выполненные.
- Очистка выполненных задач.
- Сохранение состояния задач в `localStorage`.
- Базовая автоматическая проверка в CI.
- Unit-тесты критичной бизнес-логики задач.

## Не входит в MVP (Non-goals)

- Авторизация и аккаунты пользователей.
- Серверное API и постоянное хранение данных в БД.
- Совместная работа нескольких пользователей.
- Мобильные приложения (iOS/Android).
- Drag-and-drop сортировка задач.
- Категории, теги, приоритеты, дедлайны задач.
- Уведомления (email, push, in-app).
- Многоязычность интерфейса.
- Расширенная аналитика и продуктовая телеметрия.
- Полноценный дизайн-системный UI kit.

## Ограничения v1.0.0

- Все данные хранятся только локально в браузере текущего пользователя.
- При очистке `localStorage` данные безвозвратно теряются.
- Поддерживается только веб-режим запуска через статический сервер.

## Критерий завершения по scope

Scope считается завершенным, когда все пункты раздела "Входит в MVP" реализованы,
а требования из раздела "Не входит в MVP" явно не включаются в релиз `v1.0.0`.
